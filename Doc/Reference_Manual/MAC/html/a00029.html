API Constants

Detailed Description

This section describes all constants of the API. The first part provides references to the IEEE standard [1] and also structures the constants into corresponding groups.

Capability Information Field

See section 7.3.1.1.2 in [1] for full description.

  • Bit 0 : Alternate PAN coordinator (WPAN_CAP_ALTPANCOORD) : set to 1 if the device is capable of becoming a PAN coordinator
  • Bit 1 : Device type (WPAN_CAP_FFD) : set to 1 if the device is an FFD, 0 if RFD
  • Bit 2 : Power source (WPAN_CAP_PWRSOURCE) : set to 1 if the device is receiving power from the alternating current mains
  • Bit 3 : Receiver on when idle (WPAN_CAP_RXONWHENIDLE) : set to 1 if the device does not disable its receiver to conserve power during idle periods.
  • Bit 4-5 : Reserved : always set to 0
  • Bit 6 : Security capability : set to 1 if the device is capable of sending and receiving MAC frames secured
  • Bit 7 : Allocate address (WPAN_CAP_ALLOCADDRESS) : set to 1 if the device wishes the coordinator to allocate a short address as a result of the association procedure.

Address Mode

See section 7.1.1.1.1 in [1] for full description.

  • 0x00 : no address (WPAN_ADDRMODE_NONE)
  • 0x01 : reserved,
  • 0x02 : 16 bit short address (WPAN_ADDRMODE_SHORT)
  • 0x03 : 64 bit extended address (WPAN_ADDRMODE_LONG)

Disassociation Reason Codes

See section 7.3.1.3.2 in [1] for full description.

  • 0x00 : Reserved.
  • 0x01 : The coordinator wishes the device to leave the PAN (WPAN_DISASSOC_BYPARENT).
  • 0x02 : The device wishes to leave the PAN (WPAN_DISASSOC_BYCHILD).
  • 0x03...0x7F : Reserved.
  • 0x80...0xFF : Reserved for MAC primitive enumeration values.

Association Status Field

See section 7.3.1.2.3 in [1] for full description.

  • 0x00 : Association successful (ASSOCIATION_SUCCESSFUL).
  • 0x01 : PAN at capacity (PAN_AT_CAPACITY).
  • 0x02 : PAN access denied (PAN_ACCESS_DENIED).
  • 0x03...0x7F Reserved (ASSOCIATION_RESERVED).
  • 0x80...0xFF Reserved for MAC primitive enumeration values.

Logical Channel

See section 6.1.2 in [1] for full description. The logical channel is an integer which depends on the RF frequency band used. For 2.4 GHz (BAND_2400) the channels are 11 to 26.

Channel Assignments and Numbering

See section 6.1.2 in [1] for full description of channel assignment and numbering. The scan channel mask is a 32 bit integer, where each bit is assigned to a channel:

  • Bit 0 : Channel 0 for 868 MHz transceivers
  • Bit 1...10 : Channel 1...10 for 900 MHz transceivers
  • Bit 11...26 : Channel 11...26 for 2.4 GHz transceivers (BAND_2400)
  • Bit 27...32 : reserved

Time in Symbol Periods

The symbol period for the 2.4 GHz band (BAND_2400) is 16 us. 4 bits are transmitted per symbol.

Scan Types

See section 7.1.11.1.1 in [1] for full description.

  • 0x00 = ED scan, FFD only (MLME_SCAN_TYPE_ED).
  • 0x01 = active scan, FFD only (MLME_SCAN_TYPE_ACTIVE).
  • 0x02 = passive scan (MLME_SCAN_TYPE_PASSIVE).
  • 0x03 = orphan scan (MLME_SCAN_TYPE_ORPHAN).

Beacon and Superframe Order

See section 7.1.14.1.1 in [1] for full description.

The Beacon Order (BO) describes how often the beacon is to be transmitted. The BO can be used to calculate the Beacon Interval (BI) as follows:

for 0<=BO<=14, BI = aBaseSuperframeDuration * (2^BO) symbols.

If BO = 15, the coordinator will not transmit a beacon, and the SuperframeOrder parameter value is ignored.

The Superframe Order (SO) is an Integer with allowed values from 0 to BO (or 15) and is the length of the active portion of the superframe which includes the beacon frame. The SO can be used to calculate the Superframe Duration (SD) as follows: for 0<=SO<=BO<=14, SD = aBaseSuperframeDuration * (2^SO) symbols.

If SO=15, the superframe will not be active after the beacon.

Pending Address Specification

See section 7.2.2.1.6 in [1] for full description.

From the pending address specification value, the number of pending addresses in the beacon.

  • Bit 0 ...2 : Number of short addresses pending
  • Bit 3 : Reserved
  • Bit 4 ...6 : Number of extended addresses pending
  • Bit 7 : Reserved

Super Frame Specification

See section 7.2.2.1.2 in [1] for full description.

The super frame specification is a 16 bit value, which is encoded according to this list:

  • Bit 0...3 : Beacon order
  • Bit 4...7 : Superframe order
  • Bit 8...11 : Final CAP slot
  • Bit 12 : Battery life extension
  • Bit 13 : Reserved
  • Bit 14 : PAN coordinator
  • Bit 15 : Association permit

TX Options

See section 7.1.1.1.1 in [1] for full description.

The TxOptions parameter is a bitwise OR of one or more of the following:

  • Bit 0 : acknowledged transmission (WPAN_TXOPT_ACK).
  • Bit 1 : GTS transmission.
  • Bit 2 : indirect transmission (WPAN_TXOPT_INDIRECT, WPAN_TXOPT_INDIRECT_ACK).
  • Bit 3 : security enabled transmission.
  • Bit 4 ... 7 : defined as 0

If no option is set, the macro (WPAN_TXOPT_OFF) can be used.

#define WPAN_CAP_ALLOCADDRESS   (0x80)

Flag value for capability information field (see wpan_mlme_associate_req()). The allocate address subfield shall be set if the device wishes the coordinator to allocate a short address as a result of the association procedure. If this subfield is set to 0, the special short address of 0xfffe shall be allocated to the device and returned through the association response command. In this case, the device shall communicate on the PAN using only its 64 bit extended address.

#define WPAN_CAP_ALTPANCOORD   (0x01)

Flag value for capability information field (see wpan_mlme_associate_req()). The alternate PAN coordinator subfield shall be set if the device is capable of becoming a PAN coordinator. Otherwise, the alternate PAN coordinator subfield shall be set to 0.

#define WPAN_CAP_FFD   (0x02)

Flag value for capability information field (see wpan_mlme_associate_req()). The device type subfield shall be set if the device is an FFD. Otherwise, the device type subfield shall be set to 0 to indicate an RFD.

#define WPAN_CAP_PWRSOURCE   (0x04)

Flag value for capability information field (see wpan_mlme_associate_req()). The power source subfield shall be set if the device is receiving power from the alternating current mains. Otherwise, the power source subfield shall be set to 0.
