Microcontroller Wireless Solutions


General Information

Overview

The IEEE standard 802.15.4 defines the protocol and compatible interconnection for data communication devices using low data rate, low power and low complexity, short-range radio frequency (RF) transmissions in a wireless personal area network (WPAN). Atmel developed the MAC stack software for different target platforms (microcontroller and board) and RF transceivers based on a new architecture allowing easy portability across various Atmel platforms and transceivers, and configurability to improve resource usage. This section is structured into:

  • Scope
  • Build Procedure

For more information please check the lastest release notes.

For detailed information also refer to the AVR2025 IEEE 802.15.4 MAC Software Package User Guide.

Scope

The scope of this document is to provide a description for:

  • MAC API General Functions, MAC API Request and Response Functions, and MAC API Callback Functions (Confirm and Indication Functions),
  • TAL API Functions,
  • PAL API Functions,
  • TFA API Functions,
  • SAL API Functions,
  • STB API Functions,
  • Queue and Buffer Management Functions,
  • MAC Constants, and
  • API Constants.

Implementation and design details are out of scope.

Build Procedure

Besides source code file (.h and .c), the MAC package is shipped with
  • Makefiles
  • AVR Studio Project files; i.e. .aps
  • IAR Embedded Workbench files; i.e. .eww
  • batch files.

Depending on the involved tools, different ways to build a certain example application are supported:

  • Executing Makefiles Directly
  • Executing Makefiles from AVR Studio
  • Build Process with IAR EWW
  • Batch File Build

Executing Makefiles Directly

If a build of an example application is required, the user can run "make" with the appropriated "Makefile". If the build is based on all source code files, the following example shows how to build an example application.

Example: Build MAC application using all source code files 

cd AVR2025\Applications\MAC_Examples\App_1_Nobeacon\Coordinator\AT86RF230B_ATMEGA1281_REB_2_3_STK500_STK501\GCC
make -f Makefile_Nobeacon_FFD_all_files

Executing Makefiles from AVR Studio

The build process can also be executed using the AVR Studio. After opening the AVR Studio project, e.g. 
AVR2025\Applications\MAC_Examples\App_1_Nobeacon\Coordinator\AT86RF230B_ATMEGA1281_REB_2_3_STK500_STK501\Coordinator.aps

Execute from the AVR Studio menu "Build" -> "Build All". Now the AVR Studio starts executing the Makefile.

Build Process with IAR EWW

Using the same example application, the IAR EWW can create the build. Open the IAR EWW with 
AVR2025\Applications\MAC_Examples\App_1_Nobeacon\Coordinator\AT86RF230B_ATMEGA1281_REB_2_3_STK500_STK501\Coordinator.eww

Execute from the IAR EWW menu "Project" -> "Rebuild All". Now the IAR EWW starts executing the Makefile.

Batch File Build

The MAC package contains two directories with batch scripts to recompile all example applications at once without running all single makefiles.

The corresponding batch files are located in the directories 

Build\Build_MAC\GCC
Build\Build_MAC\IAR

Using WinAVR based tools, change to 

Build\Build_MAC\GCC\AVR
and execute the following script to re-compile all example applications: 
build_all_example_apps.bat

Or using IAR based tools, change to 

Build\Build_MAC\IAR\AVR
and execute the following script to re-compile all example applications: 
iar_build_all_example_apps.bat

Note:
This may take some time - even some hours.
Due to some limitations of the used tools it is recommended to use a short name for the top level directory path, like: 
C:\Atmel\AVR2025\
