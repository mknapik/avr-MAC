Microcontroller Wireless Solutions


RCB230 V3.2 with ATmega1281 and AT86RF230 based on Sensor Terminal board

Detailed Description

Introduction

This section describes the Board Configuration for RCB_3_2_SENS_TERM_BOARD with ATmega1281 and AT86RF230B.

The board RCB_3_2_SENS_TERM_BOARD with ATmega1281 and AT86RF230B provides an IEEE 802.15.4 dual chip solution for the 2.4 GHz band.

For more information about this board see the AVR2025 MAC User Guide.

For more information about the complete API for all software layers including PAL see the AVR2025: IEEE 802.15.4-2006 MAC Reference Manual.

Transceiver-MCU interface

The transceiver interfaces the MCU via SPI and a number of GPIOs. This is shown in the picture below:

   ------------------+     +----------------
       ATmega1281    |     |    AT86RF230
                     | SPI |
       +----------------------------+
          PB0 (SS)   | --> | SEL
          PB1 (SCK)  | --> | SCK
          PB2 (MOSI) | --> | MOSI
          PB3 (MISO) | <-- | MISO
       +----------------------------+
                     |     |
          PB4        | --> | SLP_TR
          PB5        | --> | RST
                     |     |
          PD4 (ICP1) | <-- | IRQ (see TRX_IRQ and IRQ generation)
                     |     |
          PD6 (T1)   | <-- | CLKM (see Timer source)
   ------------------+     +----------------

Default clock speed (F_CPU)

The default clock speed is defined by F_CPU and is currently 8MHz. The CPU clock source is selected in mcu_clock_init() in file pal.c.

IRQ generation

The AT86RF230 provides only one transceiver interrupt. The IRQ pin from AT86RF230 is connected to the ICP1 pin of ATmega1281. The ICP1 pin acts as an input capture pin for Timer/Counter1 and is interrupt capable. The corresponding interrupt vector is defined in TRX_IRQ in pal_config.h and used in the ISR for the transceiver main interrupt (TRX_IRQ_vect) in the corresponding file pal_irq.c.

Timer usage

This board currently uses Timer 1 as system clock and hardware timer. Additionally Timer 2 is used during RC oscillator calibration (see function pal_calibrate_rc_osc() in the corresponding file pal_board.c).

Timer source

The AVR2025 MAC requires a 1MHz timer clock source, since the MAC timers implemented in pal_timer.c provide a timer resolution of 1us.

  • Timer source during periods where the transceiver is active:

    The CLKM from AT86RF230 is connected to T1 pin of ATmega1281 thus acting as Timer/Counter1 Clock Input (i.e. clock source) during time periods, where the transceiver is awake (see macro TIMER_SRC_DURING_TRX_AWAKE). The clock speed of the timer is 1MHz, since the transceiver clock is used directly.

  • Timer source during periods where the transceiver is sleeping:

    During periods where the transceiver is sleeping (and thus the CLKM is not available), the timer source is switched to the internal RC oscillator (see macro TIMER_SRC_DURING_TRX_SLEEP). The clock speed of the timer is also 1MHz, since the timer clock is derived from the current value of the system clock (F_CPU, default is 8MHz) and the corresponding prescaler.

Timestamping

Whenever an interrupt is asserted by the transceiver, the current timestamp is automatically latched into the timestamp register, which is defined by TIME_STAMP_REGISTER. For this board the timestamp register is the Input Capture Register of Timer 1 (ICR1).
The function for reading the current timestamp (pal_trx_read_timestamp()) is located in the corresponding file pal_board.c.

LEDs

This board provides 2 LEDs that are accessible via external memory. See LED_XRAM_ADDRESS and other corresponding defines.

Buttons

This board provides 1 button that is accessible via external memory. See BUTTON_XRAM_ADDRESS.

SIO (Serial I/O)

This board provides USB support for USB0. See USB_PORT and other corresponding defines.

External EEPROM

This board does not provide an external EEPROM, i.e. the IEEE address of the plaform needs to be stored in the internal EEPROM of the MCU.
Function pal_ps_get() (see pal.c) shall only be called with parameter ps_type set to INTERN_EEPROM.
Make sure that an appropiate IEEE address is stored in the internal EEPROM.

Fuse settings

The following fuse settings are recommended:
  • LF: 0xE2
  • HF: 0x91
  • EF: 0xFE

Make sure that the fuse CKDIV8 is NOT set.
